<p>在学习开源代码的时候，经常会遇到<code>__attribute__</code>的相关使用，但是在使用的时候又会忘记，所以用这篇文章总结下iOS开发中经常使用的attribute。<br><code>__attribute__</code>其实是GNU C的一种机制，本质是一个编译器的指令，在声明的时候可以提供一些属性，在编译阶段起作用，来做多样化的错误检查和高级优化。可以分别用于设置函数属性、变量属性和类型属性，语法一般为: <code>__attribute__(xx)</code>。下面分别介绍下这些属性。   </p>
<h4 id="函数属性"><a href="#函数属性" class="headerlink" title="函数属性"></a>函数属性</h4><p>函数属性可以帮助开发者把一些特性添加到函数声明中，从而可以使编译器在错误检查方面的功能更强大。   </p>
<p><strong>format</strong><br>语法为<code>__attribute__((format(__NSString__, F, A)))</code>，可以给被声明的函数加上类似printf或者scanf的特征，它可以使编译器检查函数声明和函数实际调用参数之间的格式化字符串是否匹配。<code>format (archetype, m, n)</code>，第一个参数传递archetype指定为哪种类型，string-index指定格式化字符串的位置，n指定可变参数检查开始的位置。例如:<br><figure class="highlight plain"><pre><span class="code"><pre><span class="line">#define NS_FORMAT_FUNCTION(F,A) __attribute__((format(__NSString__, F, A)))  </span><br><span class="line">FOUNDATION_EXPORT void NSLog(NSString *format, ...) NS_FORMAT_FUNCTION(1,2);</span><br><span class="line">extern void myPrint(const char *format,...)__attribute__((format(printf,1,2)));</span><br></pre></td></tr></table></figure></p>
<p><strong>objc_designated_initializer</strong><br>指定内部实现的初始化方法，系统宏<code>NS_DESIGNATED_INITIALIZER</code>展开即为该指令，语法为<code>__attribute__((objc_designated_initializer))</code>。例如：<br><figure class="highlight plain"><table><tr><td class="code"><pre><span class="line">- (instancetype)initNoDesignated ;</span><br><span class="line">- (instancetype)initNoDesignated1 NS_DESIGNATED_INITIALIZER;</span><br></pre></td></tr></table></figure></p>
<p>当一个类存在方法带有NS_DESIGNATED_INITIALIZER属性时，它的NS_DESIGNATED_INITIALIZER方法必须调用super的NS_DESIGNATED_INITIALIZER方法。它的其他方法（非NS_DESIGNATED_INITIALIZER）只能调用self的方法初始化。</p>
<p><strong>objc_requires_super</strong><br>表示子类重写当前类的方法时，必须要调用super函数，否则会有警告。语法为<code>__attribute__((objc_requires_super))</code>，例如：<br><code>- (void)fatherMethod __attribute__((objc_requires_super));</code>   </p>
<a id="more"></a>  
<p><strong>constructor/destructor</strong><br>表明该函数在mian函数被调用前/后调用，语法为<code>__attribute__((constructor/destructor))</code>。例如：<br><figure class="highlight plain"><table><tr><td class="code"><pre><span class="line">static __attribute__((constructor)) void before() &#123;</span><br><span class="line">    printf(&quot;Hello&quot;);</span><br><span class="line">&#125;</span><br><span class="line">static __attribute__((destructor)) void after() &#123;</span><br><span class="line">    printf(&quot; World!&quot;);</span><br><span class="line">&#125;</span><br><span class="line">int main(int argc, const char * argv[]) &#123;</span><br><span class="line">    printf(&quot;1&quot;);</span><br><span class="line">    return 0;</span><br><span class="line">&#125;</span><br></pre></td></tr></table></figure></p>
<p>constructor和<code>load</code>方法都是在main函数执行前调用，但是<code>load</code>更早一些。</p>
<p><strong>warn_unused_result</strong><br>指明一定要使用函数的返回值，否则会有警告，语法为<code>__attribute__((warn_unused_result))</code>。例如：<br><figure class="highlight plain"><table><tr><td class="code"><pre><span class="line">- (int)mustUseReturnValue __attribute__((warn_unused_result)) &#123;</span><br><span class="line">    return 0;</span><br><span class="line">&#125;</span><br></pre></td></tr></table></figure></p>
<p><strong>nonnull</strong><br>编译器对参数进行NULL检查，参数类型必须是指针类型，语法为<code>__attribute__((nonnull (x)))</code>，x指第x参数不能为空。例如：<br><figure class="highlight plain"><table><tr><td class="code"><pre><span class="line">- (int)addNum1:(int *)num1 num2:(int *)num2  __attribute__((nonnull (1)))&#123;</span><br><span class="line">    return  *num1 + *num2;</span><br><span class="line">&#125;</span><br></pre></td></tr></table></figure></p>
<p>类似于iOS系统宏NS_ASSUME_NONNULL(参考最后)。  </p>
<p><strong>sentinel</strong><br>指明变参函数需要nil作为边界，语法为<code>__attribute__((sentinel))</code>，例如：<br><figure class="highlight plain"><table><tr><td class="gutter"><pre><span class="line">1</span><br></pre></td><td class="code"><pre><span class="line">- (void)endWithNil:(nonnull id)first, ... __attribute__((sentinel));</span><br></pre></td></tr></table></figure></p>
<p><strong>noreturn</strong><br>通知编译器当前函数允许不返回值。当遇到类似函数还未运行到return语句就需要退出来的情况，该属性可以避免出现错误信息。语法为<code>__attribute__((noreturn))</code>。<br>例如：<br><figure class="highlight plain"><table><td class="code"><pre><span class="line">//AFURLConnectionOperation.m</span><br><span class="line">+ (void) __attribute__((noreturn)) networkRequestThreadEntryPoint:(id)__unused object &#123;</span><br><span class="line">    do &#123;</span><br><span class="line">        @autoreleasepool &#123;</span><br><span class="line">            [[NSRunLoop currentRunLoop] run];</span><br><span class="line">        &#125;</span><br><span class="line">    &#125; while (YES);</span><br><span class="line">&#125;</span><br></pre></td></tr></table></figure></p>
<p><strong>always_inline/noinline</strong><br>表示函数强制内联，语法为<code>__attribute__((always_inline/noinline))</code>。例如：<code>void test(int a) __attribute__((always_inline));</code>。<br>编译器编译内联函数时会直接将函数插入到调用的地方，所以没有普通函数调用时的额外开销(压栈/跳转/返回)。但是是否形成内联函数，需要看编译器对该函数定义的具体处理，编译器可能会拒绝内联请求。</p>
<p><strong>unavailable</strong><br>指明该方法不可用，系统宏<code>NS_UNAVAILABLE</code>的展开即为该指令，语法为<code>__attribute__((unavailable))</code>。例如：<br><figure class="highlight plain"><table><tr><td class="code"><pre><span class="line">#define NS_AUTOMATED_REFCOUNT_UNAVAILABLE __attribute__((unavailable(&quot;not available in automatic reference counting mode&quot;)))  </span><br><span class="line">- (instancetype)init NS_UNAVAILABLE;</span><br><span class="line">- (void)method11 __attribute__((unavailable(&quot;不可使用&quot;)));</span><br></pre></td></tr></table></figure></p>
<p><strong>deprecated</strong><br>表示当前方法/属性不推荐使用，编译时会报过时警告，系统宏<code>DEPRECATED_ATTRIBUTE</code>、<code>NS_DEPRECATED</code>的展开即为该指令，语法为<code>__attribute__((deprecated))</code>，deprecated后可跟注释。例如：<br><figure class="highlight plain"><table><tr><td class="code"><pre><span class="line">- (void)method1 __attribute__((deprecated(&quot;使用#method2&quot;)));</span><br><span class="line">- (void)method2 DEPRECATED_ATTRIBUTE; //DEPRECATED_ATTRIBUTE是系统的宏</span><br></pre></td></tr></table></figure></p>
<p><strong>enable_if</strong><br>用来实现C函数参数的静态检查，例如：<br><figure class="highlight plain"><table><tr><td class="code"><pre><span class="line">static void printValidAge(int age)</span><br><span class="line">__attribute__((enable_if(age &gt; 0, &quot;当前参数有问题～&quot;))) &#123;</span><br><span class="line">    printf(&quot;%d&quot;, age);</span><br><span class="line">&#125;</span><br></pre></td></tr></table></figure></p>
<p>在函数声明中，可以同时使用多个属性，用<code>,</code>隔开即可。</p>
<h4 id="类型-变量属性"><a href="#类型-变量属性" class="headerlink" title="类型/变量属性"></a>类型/变量属性</h4><p><strong>objc_subclassing_restricted</strong><br>指明当前类型不能有子类，相当于final关键字，语法为<code>__attribute__((objc_subclassing_restricted))</code>。例如：<br><figure class="highlight plain"><table><tr><td class="gutter"><pre><span class="line">1</span><br><span class="line">2</span><br></pre></td><td class="code"><pre><span class="line">__attribute__((objc_subclassing_restricted))</span><br><span class="line">@interface FinalObject : NSObject</span><br></pre></td></tr></table></figure></p>
<p><strong>objc_root_class</strong><br>表示当前类是一个根类，例如NSObject，NSProxy等。例如：<br><figure class="highlight plain"><table><tr><td class="gutter"><pre><span class="line">1</span><br><span class="line">2</span><br><span class="line">3</span><br><span class="line">4</span><br><span class="line">5</span><br><span class="line">6</span><br></pre></td><td class="code"><pre><span class="line">//NSObject</span><br><span class="line">__OSX_AVAILABLE_STARTING(__MAC_10_0, __IPHONE_2_0)</span><br><span class="line">OBJC_ROOT_CLASS</span><br><span class="line">@interface NSObject &lt;NSObject&gt; &#123;</span><br><span class="line">    Class isa  OBJC_ISA_AVAILABILITY;</span><br><span class="line">&#125;</span><br></pre></td></tr></table></figure></p>
<p><strong>NSObject</strong><br>CFDictionaryRef属于CoreFoundation框架的,也就是非OC对象,加上attribute((NSObject))后,myDictionary的内存管理会被当做OC对象来对待。例如：<br><code>@property (nonatomic,strong) __attribute__((NSObject)) CFDictionaryRef myDictionary;</code></p>
<p><strong>objc_runtime_name</strong><br>可以修改类/协议在runtime中的名称，语法为<code>__attribute__((objc_runtime_name(&quot;xxx&quot;)))</code>，可以使用该特性做代码混淆。例如：<br><figure class="highlight plain"><table><tr><td class="code"><pre><span class="line">__attribute__((objc_runtime_name(&quot;NewFather&quot;)))</span><br><span class="line">@interface Father : NSObject</span><br><span class="line">@end</span><br></pre></td></tr></table></figure></p>
<p><strong>unused</strong><br>表示变量即使不使用，也不会产生警告，语法为<code>__attribute__((unused))</code>。例如：<code>NSInteger i __attribute__((unused)) = 10;</code>。   </p>
<p><strong>objc_boxable</strong><br>可以将struct/union类型包装成NSValue对象，语法为<code>__attribute__((objc_boxable))</code>。例如：<br><figure class="highlight plain"><table><tr><td class="code"><pre><span class="line">typedef struct __attribute__((objc_boxable)) &#123;</span><br><span class="line">    CGFloat x, y, width, height;</span><br><span class="line">&#125; XXRect;</span><br></pre></td></tr></table></figure></p>
<p>这样就可以这样使用结构体了：<code>XXRect rect2 = {1, 2, 3, 4}; NSValue *value2 = @(rect2);</code>。     </p>
<p><strong>aligned</strong><br>表示所作用的结构陈冠对齐在n字节自然边界上，语法为<code>__attribute__((aligned(x)))</code>。例如：<br><figure class="highlight plain"><table><tr><td class="code"><pre><span class="line">typedef struct</span><br><span class="line">&#123;</span><br><span class="line">    char  member1;</span><br><span class="line">    int   member2;</span><br><span class="line">    short member3;</span><br><span class="line">&#125;__attribute__ ((aligned (8))) Family;</span><br></pre></td></tr></table></figure></p>
<p><strong>packed</strong><br>指定的结构结构体按照一字节对齐，语法为<code>__attribute__((packed(x)))</code>。例如：<br><figure class="highlight plain"><table><tr><td class="gutter"><pre><span class="line">1</span><br><span class="line">2</span><br><span class="line">3</span><br><span class="line">4</span><br><span class="line">5</span><br><span class="line">6</span><br></pre></td><td class="code"><pre><span class="line">typedef struct &#123;</span><br><span class="line">    char    version;</span><br><span class="line">    int16_t sid;</span><br><span class="line">    int32_t len;</span><br><span class="line">    int64_t time;</span><br><span class="line">&#125;__attribute__ ((packed)) Header;</span><br></pre></td></tr></table></figure></p>
<p>常用语与协议相关的网络传输中。  </p>
<p><strong>cleanup</strong><br>作用于变量，当变量的作用域结束是，会调用指定的函数，语法为<code>__attribute__((cleanup))</code>。该属性可以作用于变量，block，例如：<br><figure class="highlight plain"><table><tr><td class="code"><pre><span class="line">- (void)createXcode&#123;</span><br><span class="line">    NSObject *xcode __attribute__((cleanup(xcodeCleanUp))) = [NSObject new];</span><br><span class="line">    NSLog(@&quot;%@&quot;,xcode);</span><br><span class="line">&#125;</span><br><span class="line"></span><br><span class="line">// 指定一个cleanup方法，注意入参是所修饰变量的地址，类型要一样</span><br><span class="line">// 对于指向objc对象的指针(id *)，如果不强制声明__strong默认是__autoreleasing，会造成类型不匹配  </span><br><span class="line">static void xcodeCleanUp(__strong NSObject **xcode)&#123;</span><br><span class="line">    NSLog(@&quot;cleanUp call %@&quot;,*xcode);</span><br><span class="line">&#125;</span><br></pre></td></tr></table></figure></p>
<p>相关的示例，可以看下<a href="http://blog.sunnyxx.com/2014/09/15/objc-attribute-cleanup/" target="_blank" rel="noopener">黑魔法attribute((cleanup))</a></p>
<h4 id="Clang-Attributes"><a href="#Clang-Attributes" class="headerlink" title="Clang Attributes"></a>Clang Attributes</h4><p>Clang Attributes是Clang提供的一种源码注解，方便开发者向编译器表达某种要求，</p>
<p><strong>availability</strong><br>指明API(变量/方法)版本的变更，语法为<code>__attribute__((availability(macosx,introduced=m,deprecated=n)))</code>，m为引入的版本，n为废弃版本，在Cocoa的头文件中可以经常见到。例如：<br><figure class="highlight plain"><table><tr><td class="gutter"><pre><span class="line">1</span><br></pre></td><td class="code"><pre><span class="line">#define CF_DEPRECATED_IOS(_iosIntro, _iosDep, ...)    __attribute__((availability(ios,unavailable,introduced=_iosIntro,deprecated=_iosDep,message=&quot;&quot; __VA_ARGS__)))</span><br></pre></td></tr></table></figure></p>
<p>其他的参数为: obsoleted为移除的版本，unavailable为不可用，message为警告信息。系统宏<code>NS_AVAILABLE</code>、<code>NS_CLASS_AVAILABLE</code>的展开即为该属性。  </p>
<p><strong>overloadable</strong><br>在C的函数上实现像java一样方法重载，语法为<code>__attribute__((overloadable))</code>，例如：<br><figure class="highlight plain"><table><tr><td class="code"><pre><span class="line">__attribute__((overloadable)) void add(int num)&#123;</span><br><span class="line">    NSLog(@&quot;Add Int %i&quot;,num);</span><br><span class="line">&#125;</span><br><span class="line"></span><br><span class="line">__attribute__((overloadable)) void add(NSString * num)&#123;</span><br><span class="line">    NSLog(@&quot;Add NSString %@&quot;,num);</span><br><span class="line">&#125;</span><br><span class="line"></span><br><span class="line">__attribute__((overloadable)) void add(NSNumber * num)&#123;</span><br><span class="line">    NSLog(@&quot;Add NSNumber %@&quot;,num);</span><br><span class="line">&#125;</span><br></pre></td></tr></table></figure></p>
<h4 id="iOS中的宏"><a href="#iOS中的宏" class="headerlink" title="iOS中的宏"></a>iOS中的宏</h4><h5 id="系统宏"><a href="#系统宏" class="headerlink" title="系统宏"></a><strong>系统宏</strong></h5><p>a. NS_ASSUME_NONNULL<br>iOS的系统宏<code>NS_ASSUME_NONNULL_BEGIN</code>,<code>NS_ASSUME_NONNULL_END</code>，用该宏把整个类包起来，表明当前类的所有指针类型不能显示的置为nil。<br><figure class="highlight plain"><table><td class="code"><pre><span class="line">NS_ASSUME_NONNULL_BEGIN</span><br><span class="line">@interface Food : NSObject</span><br><span class="line">@property (nonatomic, copy) NSString *name;</span><br><span class="line">+ (instancetype)FoodWithName:(NSString *)name;</span><br><span class="line">@end</span><br><span class="line">NS_ASSUME_NONNULL_END</span><br></pre></td></tr></table></figure></p>
<p>通常把这个宏和nullable一起使用。  </p>
<p>b. 可用性的宏标记    </p>
<figure class="highlight plain"><table><tr><td class="code"><pre><span class="line">NS_CLASS_AVAILABLE   // 类之前</span><br><span class="line">NS_AVAILABLE(_mac, _ios) // 方法之后</span><br><span class="line">NS_DEPRECATED(_macIntro, _macDep, _iosIntro, _iosDep, ...) // 方法之后</span><br><span class="line">OBJC_SWIFT_UNAVAILABLE(&quot;use &apos;xxx&apos; instead&quot;) // 针对swift特殊说明取代它的类和方法</span><br></pre></td></tr></table></figure>
<h5 id="常量-表达式宏"><a href="#常量-表达式宏" class="headerlink" title="常量/表达式宏"></a><strong>常量/表达式宏</strong></h5><p>例如:<br><figure class="highlight plain"><table><tr><td class="code"><pre><span class="line">#define kTabBarH 49.0</span><br><span class="line">#define kScreenH [UIScreen mainScreen].bounds.size.height</span><br><span class="line">#define MKAssertMainThread() NSAssert([NSThread isMainThread], @&quot;This method must be called on the main thread&quot;)</span><br></pre></td></tr></table></figure></p>
<p><strong>函数宏</strong><br>例如：  </p>
<pre><code>#ifndef YYSYNTH_DYNAMIC_PROPERTY_OBJECT
#define YYSYNTH_DYNAMIC_PROPERTY_OBJECT(_getter_, _setter_, _association_, _type_) \
- (void)_setter_ : (_type_)object { \
    [self willChangeValueForKey:@#_getter_]; \
    objc_setAssociatedObject(self, _cmd, object, OBJC_ASSOCIATION_ ## _association_); \
    [self didChangeValueForKey:@#_getter_]; \
} \
- (_type_)_getter_ { \
    return objc_getAssociatedObject(self, @selector(_setter_:)); \
}
#endif
</code></pre><p>还有一种变参宏，例如：<br><code>#define YYAssertNil(condition, description, ...) NSAssert(!(condition), (description), ##__VA_ARGS__)</code><br>在宏定义中<code>#</code>可以连接两个字符串或者将变量转化为C语言字符串，<code>##</code>则常用与连接，另外在逗号和<strong>VA_ARGS</strong>之间的双井号，除了拼接前后文本之外，还有一个功能，那就是如果后方文本为空，那么它会将前面一个逗号吃掉。再例如：<br><code>#define weakify(object) autoreleasepool{} __weak __typeof__(object) weak##_##object = object;</code>  </p>
